# 性能优化

React 性能优化大挑战：一次理解 Immutable data 跟 shouldComponentUpdate

我在大厂写 React，学到了什么？性能优化篇

  • 尽量提升渲染无关的子组件元素到「有状态组件」的外部。
  • 在需要的情况下对 Context 进行读写分离。
  • 包装 Context 的使用，注意错误处理。
  • 组合多个 Context，优化代码。

浅谈 React 性能优化的方向

减少渲染的节点/降低渲染计算量(复杂度)

  • 0️⃣ 不要在渲染函数都进行不必要的计算
  • 1️⃣ 减少不必要的嵌套
  • 2️⃣ 虚拟列表
  • 3️⃣ 惰性渲染
  • 4️⃣ 选择合适的样式方案

避免重新渲染

  • 0️⃣ 简化 props
  • 1️⃣ 不变的事件处理器
  • 2️⃣ 不可变数据
  • 3️⃣ 简化 state
  • 4️⃣ 使用 recompose 精细化比对

精细化渲染

  • 0️⃣ 响应式数据的精细化渲染
  • 1️⃣ 不要滥用 Context

React 项目性能分析及优化

  • 减少不必要的 render，PureComponent/ShouldComponentUpdate
  • react.useMemo
  • react.useCallback
  • 谨慎使用 context
  • 小心使用 redux
  • 不可变数据，immer.js
